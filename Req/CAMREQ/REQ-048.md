# REQ-048: Replay System Data Recording

## 1. Requirement Title
Camera State Recording for Replays

## 2. Problem Statement
To implement a "Killcam" or "Replay", we need camera path data.

## 3. Proposed Solution
Record `CameraState` (Pos, Rot, FOV) into a ring buffer every frame.

## 4. Implementation Details
- **File**: `TelemetryManager.ts`
- **Logic**: `buffer.push({ t, pos, rot })`.

## 5. Code Snippets
```typescript
replayBuffer.push({ t: Date.now(), ...camera.position });
```

## 6. Expected Benefits
- **Feature**: Replays.

## 7. Dependencies
- None

## 8. Priority Level
**Low**

## 9. Estimated Complexity
**Moderate**
