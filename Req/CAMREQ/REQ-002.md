# REQ-002: Remove OrbitControls from Gameplay

## 1. Requirement Title
Remove OrbitControls Conflict in Gameplay

## 2. Problem Statement
`OrbitControls` is enabled globally in `App.tsx` with `makeDefault`. This overrides or conflicts with manual camera updates in `Player.tsx` (and the future `CameraSystem`), leading to jittery movement or fighting inputs.

## 3. Proposed Solution
Conditionally render `OrbitControls` only when in specific modes (e.g., specific debug mode or "Inspect" mode) or remove it entirely from the main `App.tsx` and place it only where needed (like a Model Viewer). For gameplay, the camera should be fully controlled by the game logic.

## 4. Implementation Details
- **File**: `src/App.tsx`
- **Change**: Wrap `OrbitControls` in a check: `{debugMode && <OrbitControls ... />}` OR remove it and rely on `CameraSystem`.

## 5. Code Snippets

**Before (`App.tsx`):**
```typescript
<OrbitControls
    makeDefault
    enableZoom={true}
    // ...
/>
```

**After (`App.tsx`):**
```typescript
{/* Only enable for dev/debug */}
{isDebugMode && <OrbitControls makeDefault />}
```

## 6. Expected Benefits
- **Stability**: Eliminates camera jitter caused by fighting controllers.
- **Control**: Ensures game logic has authoritative control over the camera.

## 7. Dependencies
- `REQ-001` (Need a replacement controller active)

## 8. Priority Level
**Critical**

## 9. Estimated Complexity
**Simple**
