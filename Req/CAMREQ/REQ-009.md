# REQ-009: Camera Transparency Occlusion (Dither Fade)

## 1. Requirement Title
Occlusion Transparency (See-Through Walls)

## 2. Problem Statement
Sometimes the camera cannot move closer (e.g. strategic view), but objects block the view. Instead of moving the camera, we should make the blocking objects transparent.

## 3. Proposed Solution
Cast a ray from Camera to Player. Collect all objects hit. Apply a custom shader or material property to make them semi-transparent or dithered.

## 4. Implementation Details
- **File**: `CameraSystem.tsx`
- **Method**: `raycaster.intersectObjects`.
- **Material**: Custom `MeshStandardMaterial` with `dither: true` and `opacity` control, or a custom shader uniform `uOccupied`.

## 5. Code Snippets
```typescript
hits.forEach(hit => {
    const mesh = hit.object as THREE.Mesh;
    if (mesh.material) mesh.material.opacity = 0.2; // Simple version
});
```

## 6. Expected Benefits
- **Visibility**: Keeps the camera stable while maintaining visibility.
- **Strategy**: Good for isometric/top-down views.

## 7. Dependencies
- Custom Shaders (optional for high quality)

## 8. Priority Level
**Medium**

## 9. Estimated Complexity
