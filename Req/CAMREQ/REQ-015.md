# REQ-015: Interpolation for Fixed Updates

## 1. Requirement Title
Physics State Interpolation

## 2. Problem Statement
Physics updates often run at a fixed rate (e.g. 60Hz), while the screen refreshes at a variable rate (e.g. 144Hz). This causes "micro-stutter" where the camera looks jerky even at high framerates.

## 3. Proposed Solution
Interpolate the target position between the Previous Physics Frame and the Current Physics Frame based on the `alpha` (accumulator) of the physics engine.

## 4. Implementation Details
- **File**: `CameraSystem.tsx`
- **Rapier**: Use `rapier`'s interpolation if available, or manually store `prevPos` and `currPos`.

## 5. Code Snippets
```typescript
// If using specific physics loop integration
const renderPos = prevPos.lerp(currPos, alpha);
camera.position.copy(renderPos);
```

## 6. Expected Benefits
- **Smoothness**: Buttery smooth movement on high refresh rate monitors.

## 7. Dependencies
- Rapier / Physics System

## 8. Priority Level
**Low** (Performance optimization)

## 9. Estimated Complexity
**Complex**
