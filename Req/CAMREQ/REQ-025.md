# REQ-025: Viewmodel Sway / Viewmodel Lag

## 1. Requirement Title
Weapon/Hand Viewmodel Sway

## 2. Problem Statement
If hands or tools are visible, they shouldn't be perfectly locked to the camera. They should lag slightly behind.

## 3. Proposed Solution
Apply inverse rotation/position smoothing to the "Hands" group relative to the camera.

## 4. Implementation Details
- **File**: `Player.tsx` (Visuals)
- **Logic**: Use `lerp` on the `HandsGroup` to follow `Camera` with a slight delay.

## 5. Code Snippets
```typescript
handsRef.current.position.lerp(targetPos, dt * 10);
handsRef.current.quaternion.slerp(targetRot, dt * 10);
```

## 6. Expected Benefits
- **Quality**: AAA feel for First Person views.

## 7. Dependencies
- First Person Assets

## 8. Priority Level
**Low**

## 9. Estimated Complexity
**Moderate**
